# Skill: repo-bootstrap

새 저장소를 시작할 때 적용하는 최소 절차.

## 목적
- 초기 구조를 빠르게 만들되 과도한 구현을 피한다.
- 운영 규칙과 문서 기준을 먼저 확립한다.

## 절차
1. 저장소 역할/범위 정의(README, docs)
2. 작업 규칙 정의(AGENTS)
3. 보고 규칙 정의(reports)
4. 빈 디렉터리 뼈대 생성(추후 확장 포인트)
5. 비범위 명시(이번 단계에서 하지 않을 일)
6. 변경 추적(커밋 단위로 기록)

- [ ] AGENTS 존재

- [ ] 초기 디렉터리 뼈대 존재
