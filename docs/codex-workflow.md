# Codex Workflow

## 1. 프로젝트 컨텍스트 (north-star 기반 요약)
- jarvis-core는 개인 자동화를 위한 오케스트레이션/기록 기준 저장소다.
- 현재 단계는 입력/상태 관리 레이어(`parser -> draft -> file writer -> memory/tasks`)에 집중한다.
- 다음 단계는 `/approve`를 거쳐 실행 레이어로 연결하는 구조를 점진적으로 확장한다.

## 2. 절대 규칙
- 최소 diff로 구현한다.
- 한 번에 한 단계만 구현한다.
- 요청 범위를 벗어나는 확장을 금지한다.
- 기존 동작을 깨뜨리지 않는다.
- 구현/판단 기준은 관련 문서 contract를 우선한다.

## 3. 작업 수행 프로토콜
1. 사전 확인
2. 구현
3. 테스트
4. Summary 출력

## 4. 사전 확인 규칙
- 관련 contract 문서를 먼저 읽는다.
- 이후 필요한 코드/연결 지점을 확인한다.
- 확인되지 않은 내용은 추측하지 않는다.

## 5. 출력 규칙 (Summary 형식)
- 수정 파일 목록
- 변경 이유
- 성공/실패 예시
- 범위 밖 작업
- 다음 단계

## 6. 금지 사항
- DB 도입 금지
- 명시적 요청 없는 구조 리팩토링 금지
- 여러 단계 동시 구현 금지
- 과도한 추상화 금지
