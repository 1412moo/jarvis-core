# Architecture (Main Repo vs Sub Repos)

## 1) 메인 저장소(jarvis-core)의 책임
- 운영 정책, 작업 규칙, 보고 형식의 관리
- 오케스트레이션 기준/계약(인터페이스 수준) 정의
- 프로젝트 간 공통 규약 및 추적 체계 유지

## 2) 서브 저장소의 책임
- 실제 도메인 기능 구현
- 서비스별 런타임 코드/테스트/배포 파이프라인 관리
- 각 서비스 내부 의사결정 및 기술 스택 최적화

## 3) 경계 원칙
- 메인 저장소는 "무엇을/어떻게 연결할지"를 정의하고,
  서브 저장소는 "실제 기능을 어떻게 구현할지"를 담당한다.
- 메인 저장소에 서비스별 구현 로직을 과도하게 넣지 않는다.
- 서브 저장소에 전사 운영 규칙 원본을 분산시키지 않는다.

## 4) 디렉터리 의도
- `docs/`: 정책/구조/방향 문서
- `reports/`: 표준 보고 템플릿/예시
- `skills/`: 반복 운영 작업 절차
- `prompts/`: 프롬프트 자산
- `orchestrator/`: 오케스트레이션 로직(추후)
- `adapters/`: 외부 연동 어댑터(추후)
- `configs/`: 공통 설정
- `memory/`: 운영 기록/지식 축적
- `scripts/`: 보조 자동화
